Project Scope

Literature Survey

Recent advances in machine learning and deep learning have revolutionized healthcare analytics. Studies demonstrate that integrating XGBoost models with physiological data can achieve prediction accuracy exceeding 92% for wellness parameters. Simultaneously, convolutional neural networks (CNNs) have shown remarkable performance in medical image analysis, with applications ranging from diagnostic imaging to physiological feature extraction.

Key research areas include:

  • Machine Learning approaches for health prediction and risk stratification
  • Deep Learning models for medical image analysis and computer vision applications
  • Integrated health information systems and data fusion techniques
  • Real-time monitoring systems for chronic disease management
  • Privacy-preserving approaches in health data processing
Identified Issues

Research Gap

While individual technologies for health analytics exist, there remains a significant gap in integrated systems that combine multiple ML/AI modalities with user-friendly interfaces. Most existing systems focus on single-domain analysis (either predictive modeling or image analysis), rather than providing comprehensive wellness assessment.

Key gaps identified:

  • Lack of unified platforms integrating multiple AI modalities for health assessment
  • Limited real-time analysis capabilities in production health systems
  • Insufficient focus on mental wellness alongside physical health metrics
  • Fragmented architecture leading to poor data flow and integration
  • Limited accessibility for end-users and healthcare practitioners
Research Problem & Solution

Research Problem

Primary Problem Statement: How can we design and implement a comprehensive AI-driven health analyzer system that seamlessly integrates machine learning-based prediction models with deep learning-based image analysis to provide real-time, actionable insights for both physical and mental wellness optimization?

Sub-problems:

  • How to preprocess and normalize heterogeneous health data from multiple sources?
  • What ML/DL architectures best balance accuracy with real-time performance?
  • How to design a user interface that makes complex AI outputs accessible?
  • How to ensure data security and privacy compliance in health systems?
  • How to evaluate and validate predictions across diverse user populations?
Goals

Research Objectives

Primary Objective

Develop an integrated AI health analyzer combining ML prediction and DL image analysis for real-time wellness optimization with >90% accuracy in health metric prediction.

Secondary Objectives

Design modular architecture enabling independent optimization of each AI component while maintaining seamless integration and data flow across the system.

Tertiary Objectives

Create user-friendly interface translating complex AI outputs into actionable insights for both healthcare professionals and individual users.

Architecture

System Components

Well360 has three independent ML pipelines sharing one FastAPI backend, one SQLAlchemy database, and one JWT auth layer. Each module exposes dedicated REST endpoints and persists results in its own database models.

Hydration Module

💧

Two prediction pipelines: form-based XGBoost prediction and lip-image MobileNetV2 CNN analysis.

  • Form Pipeline: 17-field health form → 20+ engineered features (BMI, Hydration Index, Heat Index, Water Deficit) → XGBoost regressor (liters) + classifier (risk level)
  • Image Pipeline: Base64 lip selfie → MediaPipe lip crop → Lip validator → MobileNetV2 CNN → Dehydrate/Normal + confidence score
  • XAI: SHAP values for form predictions; Grad-CAM heatmaps for lip image analysis
  • Endpoints: POST /predict/form · POST /predict/lip · GET /history/hydration · GET /history/lip-trends · GET /tracker/dashboard · GET /weather/current

Models: xgb_regressor.pkl (~1 MB) · xgb_classifier.pkl (~1 MB) · mobilenet_lip.pth (~12 MB)

Fitness Module

🏋️

Video-based exercise recognition, real-time pose estimation, and per-rep form correction.

  • Pose Estimation: MediaPipe (complexity=1) at FRAME_SKIP=2 on 480p-capped video for real-time performance
  • Analysis: Per-rep breakdown with confidence scoring, joint angle tracking, and muscle group heatmap visualisation
  • Progress: Muscle-group radar charts tracked over 30-session cycles with personalized body-balance recommendations
  • Endpoints: POST /predict/fitness/video · GET /history/fitness · GET /progress/muscle-radar · GET /progress/timeline

Core class: FitnessVideoProcessor (fitness/api_handler.py) · Exercise XGBoost model (~2 MB)

Mental Health Module

🧠

Video emotion detection (FER/MTCNN primary + Haar cascade fallback) and Librosa audio emotion analysis.

  • Video Detection: FER + MTCNN detects 7 emotions (Angry, Disgust, Fear, Happy, Neutral, Sad, Surprise) with CLAHE + bilateral filter preprocessing; maps to Russell circumplex valence-arousal for stress assessment
  • Audio Detection: Random Forest on Librosa MFCC + energy/pitch/formant features → emotion + tone + energy level
  • Personalisation: UserEmotionProfile tracks session history; flags chronic stress (≥60% negative in last 10 sessions); rotates recommendations (no repeats per emotion per user)
  • Endpoints: POST /mental-health/video · POST /mental-health/audio · GET /mental-health/profile · GET /mental-health/history · POST /mental-health/compare

Models: emotion_model.pth (~5 MB) · best_resnet_model.pth (44 MB) · Audio RandomForest (~2 MB)
